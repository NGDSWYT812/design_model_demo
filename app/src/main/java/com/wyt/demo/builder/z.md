* 模式的定义

    * 将一个复杂对象的构建与它的表示分离，使得同样的构建过程可以创建不同的表示。

* 使用场景

    * 相同的方法，不同的执行顺序，产生不同的事件结果时；
    * 多个部件或零件，都可以装配到一个对象中，但是产生的运行结果又不相同时；
    * 产品类非常复杂，或者产品类中的调用顺序不同产生了不同的效能，这个时候使用建造者模式非常合适；


* 角色介绍

    * Product 产品类 : 产品的抽象类。
    * Builder : 抽象类， 规范产品的组建，一般是由子类实现具体的组件过程。
    * ConcreteBuilder : 具体的构建器.
    * Director : 统一组装过程(可省略)。
